# マイナンバー法改正（2023年改正・2024年施行）— 利用範囲拡大と技術的影響

## 要約

2023年6月に公布された「行政手続における特定の個人を識別するための番号の利用等に関する法律等の一部を改正する法律」（令和5年法律第48号）は、マイナンバー制度の根本的な再設計といえる内容を含む。主要な変更点は以下の4点に集約される。

1. **利用基本理念の拡大**: 「社会保障・税・災害対策」限定から行政分野全般へ
2. **準法定事務制度の導入**: 国会審議なしに省令だけで利用範囲を拡張できる構造の新設
3. **健康保険証との統合**: 2024年12月2日から新規発行終了、マイナ保険証を標準へ
4. **在外邦人・国家資格約80種への拡大**: 海外転出後も失効しないカード制度

この改正は日本のデジタル・アイデンティティインフラを「国・行政主導の集中型モデル」でさらに強化する一方、プライバシー・セキュリティ上の懸念も拡大させた。EU の eIDAS 2.0 が採用する「ユーザー主権型・分散型」とは根本的に異なるアーキテクチャ選択であり、国際標準との乖離が広がっている点にも注目が必要だ。

## 背景

### なぜ今回の改正が必要だったか

マイナンバー制度は2016年の施行以来、利用範囲を「社会保障・税・災害対策」に厳格に限定してきた。この限定は、導入時の国民の懸念（番号による広範なプロファイリング）を和らげるための政治的判断だったが、デジタル化推進の観点からは制度的な障壁となっていた。

2021年のデジタル庁創設後、政府はマイナンバーカードの普及率向上と利用拡大を重要政策に位置づけ、2022年から大規模なマイナポイント付与キャンペーンを実施した。この流れを法制度面で後押しするのが今回の改正の主目的である。

一方で、2023年に相次いで発覚した「紐付けミス問題」（公金受取口座の誤登録・健康保険証情報の誤紐付けなど4類型）が国民の信頼を大きく損なった。改正法にはミス再発防止策も一部盛り込まれているが、制度拡大の方向性は変わらなかった。

### 段階施行スケジュール

| 施行日        | 主な変更内容                                                                                                       |
| ------------- | ------------------------------------------------------------------------------------------------------------------ |
| 2024年5月27日 | 国家資格約80種・自動車登録・在留資格への利用追加。海外転出後のカード継続・在外公館での交付。かざし利用の法的根拠化 |
| 2024年12月2日 | 健康保険証の新規発行終了。マイナ保険証を基本とする仕組みへ移行。資格確認書制度開始                                 |
| 2025年5月26日 | 戸籍・住民票への氏名振り仮名記載開始（改正戸籍法・住民基本台帳法施行）                                             |
| 2026年6月頃   | マイナンバーカード券面への振り仮名記載（予定）                                                                     |

## 主要な改正内容

### 1. 利用基本理念の拡大（最重要）

改正前は「社会保障制度、税制及び災害対策に関する分野」とマイナンバー利用分野が法律上明記されていた。改正後はこれらに加え「これら以外の行政分野においても利用の推進を図る」と基本理念が変更された。

これは単なる適用範囲の拡大ではなく、**マイナンバーの位置づけそのものの転換**を意味する。従来は「限定的・特例的な番号」だったものが、行政全般で利用される汎用識別子へと性格が変わった。

### 2. 準法定事務制度の導入（構造的変化）

改正の中でも特に注目すべきは「準法定事務制度」の新設である。

「法律でマイナンバー利用が認められている事務に準ずるもので、事務の性質が同一である事務」について、主務省令の規定だけでマイナンバー利用・情報連携を可能とする制度だ。従来は利用範囲を追加するたびに別表改正（国会での立法）が必要だったが、改正後は**省令改正だけで利用範囲を拡張**できる仕組みとなった。

この変更は行政の柔軟性を高める一方、国会審議による民主的チェック機能を弱める。日本弁護士連合会は「法律による具体的な限定を欠く規制緩和」として問題視している（[日弁連会長声明、2023年3月29日](https://www.nichibenren.or.jp/document/statement/year/2023/230329.html)）。

### 3. 利用範囲の具体的拡大

2024年5月27日施行分で、以下の事務へのマイナンバー利用が追加された：

- **国家資格約80種**: 理容師・美容師・小型船舶操縦士・一級建築士などの資格に関する事務。マイナポータルを通じた資格申請時の添付書類省略が可能に
- **自動車登録**: 車検証の電子化と連動した本人確認の効率化
- **在留資格**: 外国人の在留許可等に係る事務

### 4. 健康保険証との統合

2024年12月2日から健康保険証の新規発行が終了し、**マイナ保険証（マイナンバーカードを健康保険証として利用する仕組み）が事実上の標準**となった。

| 移行スキーム | 内容                                                                         |
| ------------ | ---------------------------------------------------------------------------- |
| マイナ保険証 | マイナンバーカードを保険証として利用（2021年10月開始、2024年12月から標準化） |
| 資格確認書   | カード非保有者向け。有効期限1年の紙/プラスチック証書。更新申請が必要         |
| 移行措置     | 既発行の紙保険証は2025年12月1日まで有効（2026年7月末まで暫定延長）           |

医療機関・薬局は**オンライン資格確認システムへの対応が実質必須**となり、未対応の場合は患者の保険資格確認が困難になる。

### 5. かざし利用の法的根拠化

暗証番号入力なしでマイナンバーカードをかざすだけで利用できる「かざし利用」が法律に明記された。利用想定場面として、図書館カード・避難所受付・医療機関受付が挙げられている。PINレスでの利用はセキュリティを下げる代わりに、PINを忘れた高齢者等でも利用しやすくする実用性優先の選択だ。

### 6. 在外邦人への拡大

2024年5月27日から、海外転出者が在外公館でマイナンバーカードの新規申請・更新ができるようになり、海外転出後もカードが失効しなくなった（[外務省](https://www.mofa.go.jp/mofaj/toko/todoke/maina.html)）。海外居住者が日本の行政サービス（マイナポータル等）にアクセスできる基盤が整った。

### 7. 氏名の振り仮名・ローマ字表記

改正戸籍法・住民基本台帳法の施行により、2025年5月26日から戸籍・住民票に振り仮名が法定記載事項として追加される。「氏名として用いられる文字の読み方として一般に認められているもの」という規律も設けられ、行政上の氏名読み方に一定の標準化をもたらす（[法務省](https://www.moj.go.jp/MINJI/furigana/index.html)）。

これは国際送金・海外本人確認等での氏名ローマ字表記の標準化にも寄与し、デジタル・アイデンティティの国際相互運用性向上につながる。

## 技術的実装への影響

### 公的個人認証サービス（JPKI）の変化

改正法と並行して、JPKI インフラも大きく変化している：

- **スマホ用電子証明書**: スマートフォンの Secure Element（GP-SE）に署名用・利用者証明用の2種類の電子証明書を搭載。署名用は英数字6〜16文字の PIN（従来のカードと同等）
- **Webブラウザ連携**: 2024年3月からスマホ JPKI がWebブラウザからの本人確認に対応。専用ネイティブアプリを開発せずに公的個人認証を実装可能になった（[公的個人認証サービスポータル](https://www.jpki.go.jp/sumaho-cert/index.html)）

これにより、民間事業者がマイナンバーカードを使った本人確認サービスを実装するハードルが下がった。従来は専用アプリ開発が必要だったが、OAuth 2.0 ベースの認可フローでの組み込みが現実的になりつつある。

### マイナポータル API の現状

2024年10月時点で180以上の事業がマイナポータル API を活用している（[デジタル庁](https://digital-agency-news.digital.go.jp/articles/2024-11-28)）。提供されているAPIは11系統あり、主要なものは次のとおりだ：

| API 種別            | 取得できる情報                   |
| ------------------- | -------------------------------- |
| 医療保険情報取得API | 薬剤情報・特定健診情報・手術情報 |
| 自己情報取得API     | 所得・住民税・世帯情報           |
| 資格確認API         | 健康保険の被保険者資格           |

民間事業者にとっての現実的な課題として、デジタル庁への利用申請審査に数ヶ月を要するケースがあり、PoC 段階での開発が困難という声がある。PocketSign MynaConnect 等の中間プラットフォームが登場し、民間事業者の接続コスト低減を支援している。

### 準法定事務制度が実装者に求めること

省令改正だけで情報連携対象が追加される仕組みになったため、**民間事業者は省令の継続的なモニタリングが必要**になった。従来は法改正のたびに対応計画を立てられたが、省令改正は告示から施行まで期間が短いケースもあり、システム対応のリードタイムの問題が生じる可能性がある。

## 課題と懸念点

### プライバシー上の構造的問題

日弁連は2023年3月の会長声明で、今回の改正について以下の問題を指摘している：

- マイナンバーは「原則生涯不変の個人識別番号」であり、利用分野拡大によって「より広範な個人情報が番号に紐付けられ、漏れなく名寄せされ、プロファイリングされる危険性」が高まる
- 事前のプライバシー影響評価（PIA）手続きを行わないまま法改正を急いでいる
- 「準ずる事務」を省令で定める構造は、国会の民主的コントロールを弱める

### 紐付けミス問題と再発防止の限界

2023年に発覚した4類型の誤登録・誤紐付け事案のうち、特に深刻だったのは健康保険証情報の誤紐付けだ。保険者が氏名・生年月日・性別・住所の部分一致のみでマイナンバーと被保険者資格情報を連携していたため、同姓同名等で誤紐付けが発生した。

再発防止策としてガイドラインの策定や横断的組織の設置が行われたが、根本的な問題（照合精度・エラー検知の仕組み）が技術的に解決されたかは不明確なままだ。

### セキュリティの現状

個人情報保護委員会の令和6年度年次報告によると、マイナンバー関連の報告事案は地方公共団体等から9件、事業者から74件あった。同報告では地方公共団体に対し安全管理措置（特に教育研修・委託先管理）の継続的な改善が必要と指摘されており（[個人情報保護委員会](https://www.ppc.go.jp/aboutus/report/annual_report_2024/)）、委託先への再委託管理が不十分な事例も継続して発生している。

## eIDAS 2.0 との比較的考察

EU の eIDAS 2.0（2024年3月制定）が「ユーザー主権」「分散型ID（DID/SSI）」「選択的開示」を制度レベルで明文化しているのに対し、改正マイナンバー法は国・行政主導の集中型ID管理モデルを強化する方向性だ。

| 観点                 | 改正マイナンバー法（日本）       | eIDAS 2.0（EU）                |
| -------------------- | -------------------------------- | ------------------------------ |
| アーキテクチャ       | 集中型（国家番号＋中央DB）       | 分散型（EUDI Wallet＋DID検討） |
| ユーザーコントロール | 限定的（マイナポータルで閲覧可） | ユーザー主権を明文化           |
| 選択的開示           | 非対応（全情報開示が基本）       | Selective Disclosure を標準化  |
| 民間連携             | マイナポータル API 経由          | QEAA / PID Provider 認定制度   |
| VC/DID 対応          | 法的位置づけ未整備               | ARF で技術選択肢として列挙     |

日本では Verifiable Credentials や DID といった分散型技術の法的位置づけは未整備のまま。デジタル庁はトラストフレームワークの議論を進めているが、制度化のペースは EU と比べて大幅に遅れている。関連仕様については [OID4VP](../specs/oid4vp.md) や [SD-JWT VC](../specs/sd-jwt-vc.md) の解説も参照されたい。

## まとめ

改正マイナンバー法は、日本のデジタル・アイデンティティインフラを確実に前進させる内容を含む。健康保険証統合・スマホ JPKI 対応・国家資格での活用拡大は、行政デジタル化の具体的な成果だ。

一方で、実装者として押さえておくべき構造的リスクが3点ある：

1. **省令モニタリングの必要性**: 準法定事務制度により、利用範囲が省令改正だけで拡張されうる。システム対応のリードタイムを想定したモニタリング体制が必要
2. **照合精度の重要性**: 紐付けミス問題が示すように、マイナンバーと業務データを連携する際の照合精度設計はリスク管理上の最重要事項
3. **国際標準との乖離**: EU の eIDAS 2.0 との設計思想の差が今後も広がる可能性がある。グローバル対応を見据えたシステム設計では、将来的な Verifiable Credentials 対応を視野に入れることが望ましい
