# ID基盤が生む収益——toB・toC別の売り上げ直結機能

> ID基盤は「コスト部門」から「収益エンジン」へと変貌しつつある。認証・認可の基盤機能をどう設計・プライシングするかが、製品の競争力を左右する時代になった。

## はじめに

「ID基盤は費用がかかるインフラ」という認識は過去のものになりつつある。Auth0・WorkOS・Okta Customer Identity・Transmit Security といった主要プレイヤーが示した事実は、認証・認可・プロファイル管理の各機能が**直接的な収益ドライバー**になりうるということだ。

本稿では、ID基盤に実装することで売り上げに直結しやすい機能を **toB（企業向け）** と **toC（消費者向け）** に分けて整理する。プロダクトマネージャー・アーキテクト・ビジネス開発担当者が「何から着手するか」を判断する際の参考にしてほしい。

---

## toB：エンタープライズ市場での収益機能

### 1. Enterprise SSO（SAML / OIDC）— アップセルの王道

Enterprise SSO は「エンタープライズプラン専用機能」として位置づけることで、最も確実にARPU（顧客単価）を引き上げる機能だ。WorkOS・Auth0 いずれも、SAML/OIDC 対応 SSO をフリー・スタータープランから除外し、エンタープライズ層への誘導弁として機能させている。

- 導入効果：営業担当が「SAML対応してますか？」という問いに即答できるだけで、エンタープライズ商談の成立率が変わる
- 設計上の注意：シングルテナント型の SSO 設定画面（Organization ダッシュボード）を独立させることで、管理者の体験も差別化できる

### 2. SCIM 自動プロビジョニング — 解約防止の要

SCIM（System for Cross-domain Identity Management）は、Okta・Azure AD などのIdPとのユーザー同期を自動化する仕様だ。エンタープライズ顧客がSCIMを使い始めると、IdPとの依存関係が深まり、**解約コスト（スイッチングコスト）が劇的に上がる**。

NRR（Net Revenue Retention）120%超を達成しているSaaS企業の多くは、このSCIMを含む「ディープ・インテグレーション」を顧客定着の柱にしている。

- 売り上げへの直結ポイント：SCIM連携済み顧客のチャーンレートは、未連携顧客の数分の一に低下する傾向がある
- 実装優先度：SSO と SCIM は「セット」で提供することが多く、両者を Enterprise プランの目玉に据えるのが定石

### 3. 組織・マルチテナント管理 — B2B2C モデルの収益基盤

SaaS が「自社の顧客企業のエンドユーザー」まで認証管理するケース（B2B2C）では、テナントあたりのユーザー数・API コール数に応じた**従量課金モデル**が成立する。

重要な設計判断は以下の2点だ：

1. **Organizations（テナント）単位の管理粒度**：テナントごとのSSO設定・RBAC・監査ログを独立させることで、顧客企業の管理者に「自分のIDプラットフォームを持っている」体験を提供できる
2. **委任管理（Admin API）**：テナント管理を顧客企業のIT部門に委ねる機能は、自社サポートコストの削減にもなる

### 4. 監査ログ・コンプライアンスレポート — 規制対応での差別化

SOC 2・ISO 27001・eIDAS 2.0・金融庁ガイドラインなど、規制要件の高まりに伴い、「誰がいつ何にアクセスしたか」を証明できる監査ログは**エンタープライズ選定の必須要件**になりつつある。

- 収益モデル：監査ログの保存期間（90日・1年・5年）でプランを階層化し、長期保存を上位プランに誘導する
- 2026年注目点：eIDAS 2.0（2026年12月施行）に向けた EU 域内での準拠要件が、欧州展開を狙う企業の購買トリガーになる

### 5. Adaptive MFA / リスクベース認証 — セキュリティの商品化

MFA（多要素認証）を「オン/オフ」の設定ではなく、ログインコンテキスト（デバイス・場所・行動パターン）に応じて強度を変える**アダプティブMFA**は、CISO への訴求力が高い。

- 定量効果：アカウント乗っ取り（ATO）による企業損失は年間数百万ドル規模に上るケースもあり、リスクベース認証の導入で不正検知コストを大幅削減できる
- プライシング示唆：リスクシグナルの分析・保存・ポリシー設定を有料機能とし、基本 MFA は無料で提供するフリーミアム設計が機能する

## toC：消費者市場での収益機能

### 1. パスキー（FIDO2 / Passkeys） — コンバージョン率直結の最有力機能

パスキーは現在、toC における最も「収益直結」が実証されている認証機能だ。主要な数字を挙げる：

- **コンバージョン率 70% 向上**（Corbado 調査：パスキー導入サービス平均）
- **サインイン時間 50% 短縮**（KAYAK 事例）
- **決済離脱率 40% 削減**（ECサイト事例：パスキーによるワンタップ購入）
- **パスワードリセットサポートチケットがほぼ消滅**（TikTok・Google 等の報告）

パスキーが「コンバージョン率を上げる」のは直感に反して見えるが、理由は明快だ。パスワード入力・2段階認証の手間が消えることで、**購入・登録フローの摩擦が激減する**。特に EC・フィンテック・サブスク型サービスでのROIは即座に計測できる。

設計上のポイントとして、パスキーは**パスワードとの並列提供**から始め、段階的に移行を促すプログレッシブ移行戦略を採ることが重要だ。Auth0・Transmit Security・Hanko などの CIAM プロバイダーはいずれもこのアプローチを推奨している。

### 2. ソーシャルログイン + プログレッシブプロファイリング — 登録CVRの最大化

新規ユーザー獲得においてフォームの長さとCVRは反比例する。ソーシャルログイン（Google / Apple / LINE 等）で最低限の情報のみ取得し、その後の利用状況に応じて**段階的にプロファイルを充実させる**「プログレッシブプロファイリング」は、CIAMにおける定番の収益最大化手法だ。

- 実装指針：初回登録時に聞くのはメールアドレスのみ。電話番号・誕生日・住所は「この機能を使うために必要」という文脈でその都度収集する
- マーケティング価値：段階的に収集された属性データは、精度の高いセグメンテーションと有料プランへのアップグレード誘導に使える

### 3. Bot検知・不正アカウント対策 — 売り上げ保護の観点

不正なアカウント作成（クレデンシャルスタッフィング・バルク登録）は、消費者向けサービスにとって直接的な収益毀損だ。プロモーション乱用・不正ポイント取得・偽レビューなどがその典型例。

- 収益保護の観点：ボット対策で年間数億円規模の販促コスト流出を防いだ事例は国内でも複数存在する
- 技術的アプローチ：デバイスフィンガープリント・行動バイオメトリクス・IP レピュテーションを組み合わせた多層防御を、CIAM の認証フローに統合する

### 4. Verifiable Credentials / eKYC — 高い本人確認品質の商品化

金融・医療・不動産など、**高い本人確認品質が必要な垂直市場**では、Verifiable Credentials（VC）と連携したeKYCが新たな収益源として台頭している。

- 2025年の現状：デセントラライズド認証市場は130億ドル規模（2025年推計）まで成長し、企業KYCにおける摩擦削減・コスト削減のユースケースが具体化しつつある
- 日本の文脈：マイナンバーカード + Digital Credentials API の組み合わせが2028年以降に本格普及すると、JPKI ベースの本人確認を活用したeKYCサービスが顧客獲得コスト（CAC）を大幅に削減できる
- 収益モデル：「本人確認完了ユーザー」に対するプレミアムサービス（与信枠拡大・即時決済など）は、金融系ではARPU向上の直接手段になる

### 5. アイデンティティ分析・エンゲージメントインサイト — データとしてのID

CIAM が蓄積するログイン・セッション・デバイス・位置情報データは、**マーケティング分析の一級品**だ。

- 活用例：「パスキー登録済みユーザーはLTVが高い」「ソーシャルログイン経由ユーザーはチャーンしやすい」などの仮説検証が可能になる
- 製品化の方向性：ID 分析ダッシュボードを BIツールと連携させ、データドリブンなグロース施策を支援する機能を上位プランで提供する

## 新興領域：2026年以降に注目すべき機能

### AI エージェント認証

人間に代わって API を呼び出す AIエージェントの普及により、「エージェントが誰の権限でどのスコープまで操作できるか」を管理する**非人間アイデンティティ（NHI）管理**が急速に重要性を増している。

- toB 収益機会：AI エージェントのアイデンティティライフサイクル管理（発行・ローテーション・失効）を有料 API として提供するモデルが登場しつつある
- 関連標準：WIMSE（Workload Identity）・Agentic JWT ドラフト・IETF の委譲チェーン関連 RFC

### B2B KYC の自動化（VC × エンタープライズ）

企業間の取引における身元確認（KYB: Know Your Business）を Verifiable Credentials で自動化する動きが欧州を中心に進んでいる。eIDAS 2.0 の PID（Personal Identification Data）を法人認証に拡張するユースケースで、**コンプライアンスコストの大幅削減**が見込める。

## まとめと展望

| 軸 | 最優先機能 | 収益への効き方 |
|---|---|---|
| **toB** | Enterprise SSO + SCIM | アップセル・チャーン防止 |
| **toB** | 監査ログ・コンプライアンス | 規制対応での差別化 |
| **toC** | パスキー | コンバージョン率・離脱率 |
| **toC** | Progressive Profiling | 登録CVR・マーケティングデータ |
| **共通** | Adaptive MFA / Bot対策 | 不正損失の回避 |

私の見立てでは、**2026年現在最も ROI が明確な投資先はパスキーだ**。実装コストは中程度だが、EC・フィンテック・サブスクリプション系サービスにおけるコンバージョン改善効果は即座に計測でき、意思決定者への説明が容易だ。

toB では **Enterprise SSO の「壁」を意図的に設計すること**が重要だ。エンタープライズ商談でSSOが「あって当然」になった今、それを無料で提供するのは機会損失になりかねない。WorkOS が示した「SSO を売りにする」モデルは、中規模SaaSにとって参考になる。

ID基盤は今後、認証インフラという位置づけを超え、グロース戦略の中核に置かれるべき資産になる。何をフリーで提供し、何を有料化するか——その設計判断が、製品の中長期的な競争力を決める。
