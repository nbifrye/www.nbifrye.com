# パスキー主流化の現実：数字が語る日本の2025年

> フィッシング詐欺が89%増加した2025年、日本の金融・テック企業がパスキーへ急転換した。メルカリ1000万人・LINE Yahoo 2700万人——数字が示す「もうパスワードに戻れない」局面。

## 背景

「パスキーはまだ普及期」という認識は2025年に崩れた。

FIDO Alliance が 2025年10月に公開した **Passkey Index**（Amazon、Google、メルカリ、NTT DOCOMO、PayPal、Target、TikTokなど主要事業者のデータを集計）によれば、パスキー対象アカウントの36%がすでにパスキーを登録済みで、全サインインの26%がパスキー経由だ。

日本では同年、フィッシング詐欺報告件数が前年同期比で**89%増加**した。生成AIで作られた精巧な偽メールの横行が背景にある。この脅威の急拡大が、金融業界の意思決定を加速させた。

## 日本企業の数字

### メルカリ——2年で1000万人

メルカリは2023年3月にパスキーを導入し、2025年5月3日に**登録者1000万人突破**を達成した。社内でも「想定を超えるスピード」と評された。FIDO Passkey Index にも集計対象として名前が挙がっており、日本発のパスキー成功事例として国際的に参照される存在になっている。

### LINE Yahoo——スマホ認証の50%がパスキー

LINEヤフーはパスキーを利用するアクティブユーザーが**2700万人**に達した。スマートフォンにおいては全認証の約**50%がパスキー**によるもので、SMS ワンタイムパスワードと比較して認証速度が**2.6倍速い**という実績が出ている。

### NTT DOCOMO——2年以上、不正購入ゼロ

dアカウントのパスキー認証は全体の約**50%まで増加**。そしてdocomo Online Shopでは、パスキー導入から**2年以上にわたって不正購入が観測されていない**。これは強力な証跡だ。フィッシング耐性の効果が数字として現れている。

### KDDI——サポートコスト35%減

au ID で FIDO 認証を利用するユーザーが**1300万人**。パスワード忘れに起因するカスタマーサポートへの電話問い合わせが約**35%減少**した。セキュリティ強化とコスト削減が同時に実現した好例だ。

### 金融・証券業界——フィッシング被害を受けての必須化

証券口座の乗っ取り事件が相次いだことを受け、大和証券・みずほ証券・楽天証券が導入を進め、**野村証券は2025年11月29日からパスキー認証を必須化**した。国内でパスキーを導入済みまたは導入予定の企業は少なくとも**55社**（2024年比で約2倍）。

## グローバル文脈

グローバルな数字も補足しておく。

- **Google**：8億アカウントがパスキー利用、25億回以上のパスキーサインイン
- **Amazon**：1億7500万人がパスキー有効化、パスワード比6倍速い
- **Microsoft**：2025年5月に新規アカウントのデフォルト認証をパスキーに変更、認証数が**120%増加**
- **TikTok**：パスキー認証の成功率**97%**

Passkey Index の集計ベースでは、パスキーのサインイン成功率は**93%**（従来方式63%）、サインイン時間は**73%削減**（平均8.5秒 vs 31.2秒）、ヘルプデスクのログイン関連問い合わせが**81%減**という数字が出ている。

## 仕様面の動き——ベンダーロックインへの回答

普及を妨げてきた「パスキーを別のパスワードマネージャーに移せない」問題に対し、FIDO Alliance は **Credential Exchange Protocol（CXP）** と **Credential Exchange Format（CXF）** を策定した。

CXP は2つのクレデンシャルプロバイダー間でパスキーをセキュアに転送するプロトコル（暗号化には HPKE: RFC 9180 を使用）。開発には Apple・Google・Microsoft・Bitwarden・1Password・Dashlane・Samsung・SK Telecom が参加した。

Apple iOS 26 が最初の実装プラットフォームとなり、Bitwarden もサポートを表明している。「iCloud Keychain に閉じ込められる」という不安が解消されれば、パスキー採用への心理的障壁はさらに下がるだろう。

また NIST は 2025年7月に **SP 800-63-4** を最終版として公開。iCloud Keychain や Google Password Manager に保存された「同期パスキー（Synced Passkeys）」が **AAL2（多要素認証レベル）** を満たすと明記した。政府・規制当局のお墨付きが出たことで、企業の法務・コンプライアンス部門がブレーキを踏みにくくなった。

## 所感

日本のデータで目を引くのは「認知はしているが理解は浅い」という断層だ。ウェルスナビの2025年調査では**80%が名前を知っている**一方、**詳細を説明できるのは13%**にすぎない。

この状況は実は普及にとって悪くない。ユーザーがパスキーの仕組みを「理解しなくても使える」ことが理想であり、メルカリやdocomoの数字はまさにそれを証明している。UXが十分に洗練されれば、理解なき採用は問題にならない。

一方、課題は「デバイスを替えたときの体験」と「アカウントリカバリー」だ。新しいスマホに機種変更した際にパスキーがスムーズに引き継げるかどうか——CXP の普及がここに直接効く。

フィッシング詐欺の急増と、証券・金融業界の必須化の波が日本でのパスキー普及を引っ張っている。2026年は「パスキーがあれば安心」から「パスキーがなければ不安」という感覚が一般に広まる年になると見ている。
