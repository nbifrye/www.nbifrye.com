# /review スキル

既存コンテンツを **正確性レビュー + 校正** するためのワークフロー。

## 使うタイミング

- 既存記事の事実確認（RFC/仕様/公式ドキュメントとの整合確認）が必要なとき
- 用語ぶれ・曖昧表現・断定しすぎ表現を校正したいとき
- レビュー完了記事に `レビュー済み` タグを付与したいとき

## 手順

1. 対象記事を 1 本選ぶ（ユーザー指定がなければ任意で 1 本）。
2. 記事内の主張を抽出し、一次情報（仕様書・公式サイト）と突合する。
3. 誤り・誤解を招く表現・古い記述を修正する。
4. 記事の frontmatter に以下を設定する。
   - `tags` に `レビュー済み` を追加
   - `reviewed_at` にレビュー実施日（`YYYY-MM-DD`）
5. 必要に応じて、レビュー済みタグが UI 上で判別できるよう一覧表示ロジックも更新する。
6. `.md` を編集したら必ずフォーマットを実行する。

## 正確性レビュー基準

- 「MUST/SHOULD」などの要件レベルを一次情報どおりに扱う
- RFC が更新・補完されている場合はその関係を明記する
- BCP と元仕様の要求レベルを混同しない
- セキュリティ推奨は「仕様上の必須」と「運用上の推奨」を分けて記述する

## 出力要件

- 記事本文の修正（必要箇所のみ）
- `レビュー済み` タグ付与
- 変更内容を要約し、どの一次情報で確認したかを示す
