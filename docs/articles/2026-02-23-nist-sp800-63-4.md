# NIST SP 800-63-4 最終版が問い直す「デジタルアイデンティティ保証」のあり方

> 2025年7月に確定した SP 800-63-4 は、単なる改訂版ではなくパラダイムシフトだ。チェックリスト的な保証レベル選択から、リスクと衡平性を中心に据えた継続的マネジメントへ。

## はじめに

2025年7月31日、NIST（米国立標準技術研究所）は [Special Publication 800-63-4 Digital Identity Guidelines](https://pages.nist.gov/800-63-4/) を最終版として公開した。前版の SP 800-63-3（2017年）からおよそ8年ぶりの大幅改訂であり、IAL（Identity Assurance Level）・AAL（Authenticator Assurance Level）・FAL（Federation Assurance Level）という三本柱の枠組みは維持しながらも、その選び方・使い方の哲学を根本から刷新している。

この記事では、SP 800-63-4 の核心にある三つの変化点――DIRM フレームワーク、PAD 必須化、mDL/VC 受け入れ――を軸に、実装側が何を変えなければならないかを考察する。

## 背景：なぜ今、改訂が必要だったか

2017年版が策定された当時、SMS OTP は広く許容される多要素認証として扱われていた。しかし SIM スワップ攻撃の台頭、フィッシング耐性の議論、AI 生成によるディープフェイクを用いた本人確認突破事例の増加など、脅威の地形は激変した。

同時に、eIDAS 2.0 の進展や ISO 18013-5（mDL）の普及により、従来の「KYC = 書類アップロード + 窓口確認」という前提も揺らいでいる。デジタルウォレットが既存の物理書類と同等以上の保証を提供できる世界を、ガイドラインが想定していなかったのだ。さらに、旧来のリスク評価が「組織にとってのリスク」のみを重視し、ユーザーへの影響（アクセス不能・差別的排除・プライバシー侵害）を軽視しているという批判も根強かった。

## DIRM：チェックリストからリスクマネジメントへ

SP 800-63-4 最大の変革は **Digital Identity Risk Management（DIRM）** フレームワークの導入だ。

従来、保証レベルの選択は「業務機能とリスクカテゴリのマッピング表を参照して IAL/AAL を決定する」という静的なプロセスだった。DIRM はこれを、継続的に更新するリスク評価サイクルへと置き換える。

評価対象となるリスクは四つの軸で構造化される：

- **組織のミッション・ビジネスへの影響**（従来からあった軸）
- **個人へのリスク**（公平性・プライバシー・利便性への悪影響）
- **公共の信頼**（システム障害が社会的信頼を損なうリスク）
- **コンプライアンス**（法規制・契約上の義務）

そして組織は、IAL/AAL/FAL を選択した根拠・残留リスクの受容判断・代替手段の検討結果を **Digital Identity Acceptance Statement（DIAS）** として文書化することが必須となった。これは単なる設計書ではなく、継続的に見直し・更新する生きた文書として位置づけられる。

この変化が意味するのは、「IAL2 を満たせばいい」という思考の終わりだ。なぜ IAL2 なのか、IAL2 を選ぶことでどのユーザー層が排除されるリスクがあるか、そのトレードオフをどう受け入れるかまで、設計者は説明責任を持つ。

## PAD 必須化：ディープフェイク時代の生体認証

SP 800-63-4 が打ち出した最も即効性の高い変更の一つが、**Presentation Attack Detection（PAD）の必須要件化**だ。

遠隔での生体特性（顔・指紋など）収集を伴う本人確認プロセスは、ISO/IEC 30107-3:2023 に準拠した PAD 実装が必須となる。具体的な閾値として **IAPAR（Impostor Attack Presentation Accept Rate）< 0.07**（7%未満）が要件として明示されており、AI 生成画像・動画による攻撃への対抗措置を定量的に担保することが求められる。

この要件が影響するのは、遠隔 eKYC（顔写真 + セルフィー照合）を主要な本人確認手段として採用しているサービスだ。既存システムが ISO/IEC 30107-3 準拠の PAD を実装していない場合、あるいは IAPAR の計測・証明メカニズムを持っていない場合、IAL2 相当のサービスとして認められなくなる可能性がある。

日本においても、2025年4月施行の犯罪収益移転防止法改正（2027年完全移行）を見越した eKYC システムの刷新が業界課題となっているが、PAD 要件の国際的な収斂という観点でも NIST 基準は参照軸になるだろう。

## mDL と Verifiable Credentials の格上げ

SP 800-63-4 が宣言した三つ目の変化は、デジタルウォレット上の認証情報を「一等市民」として扱うことだ。

モバイルドライバーライセンス（ISO 18013-5 準拠の mDL）・EUDI Wallet が発行する European Digital Identity Document・W3C Verifiable Credentials などのユーザー管理型デジタルウォレットが、物理パスポートや PIV カードと**同等の最高保証レベル証拠**として認定された。

従来の SP 800-63A-3 では、IAL3（最高保証）の証拠として認められるのは REAL ID 準拠の物理書類または PIV 系スマートカードが基本だった。SP 800-63-4 では、適切な発行プロセスと技術的整合性を持つデジタル証明書類が正式にスコープに入る。

この変化の実装上の含意は大きい。EU EUDI Wallet のクロスボーダー利用、あるいは米国の mDL 普及が進む中で、SP 800-63-4 準拠を謳うフェデレーション基盤は Verifiable Presentation の検証フローを組み込む設計が求められる。OpenID4VP や DCQL といった標準が「あると便利」から「必須」に近づく転換点だ。

## AAL2 を揺るがす認証器の整理

セキュリティ実務者にとって即座に影響が出る変更として、認証器の保証レベル再編がある。

SP 800-63-4 では **SMS OTP は AAL2 相当として明示的に排除**される方向が強化された（AAL1 としても推奨されない位置づけ）。フィッシング耐性を持たない TOTP や知識ベース認証（秘密の質問）も同様に、高保証サービスでの採用根拠を失う。

一方、FIDO2/WebAuthn ベースのパスキーは AAL2 の「フィッシング耐性多要素認証」として明確に推奨される。エンタープライズ ID 基盤を構築する組織にとって、パスキー移行が単なる UX 改善ではなく **コンプライアンス要件への対応**という文脈を持つことを、SP 800-63-4 は明確にした。

## まとめと展望

SP 800-63-4 が示す方向性を一言で表すなら、「リスクと責任の内面化」だ。保証レベルは規制当局が決めるのではなく、サービス提供者が DIRM プロセスを通じて自ら判断し、DIAS として説明責任を持つ。生体認証には定量的な性能証明を要する。デジタルウォレットは物理書類と同等の存在として設計に織り込む。

日本の行政・金融・ヘルスケア分野への直接的な法的拘束力は薄いが、NIST ガイドラインは事実上の国際標準として参照される。特に政府向けシステムや海外事業者との連携が発生するサービスは、SP 800-63-4 の枠組みで自社の保証レベル選択を見直す契機として捉えるべきだろう。DIAS の考え方は、日本企業の社内セキュリティポリシー文書化にも応用できる実践的フレームワークだ。

次の注目点は実装コミュニティの反応だ。FIDO Alliance の Passkey Index、OpenID Certification、あるいは FedRAMP といった認定プログラムが SP 800-63-4 準拠要件をいつ・どう取り込むかで、市場への浸透速度が決まる。
