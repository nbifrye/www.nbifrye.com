# /bok — IAM基礎・実務ガイド執筆スキル

日本向け・初学者向けの「IAM基礎・実務ガイド（BoK）」を、
企画書に基づいて段階的に執筆・更新するためのスキル。

## 参照資料

- 企画書（原本）: `references/iam-bok-proposal.md`

## 目的

- 企画書で定義された方針・章構成・品質基準を維持しながら BoK を執筆する
- 初学者が順に読める学習導線を保つ
- 日本実務（制度・運用）と最新論点（Passkeys, NHI, AI, VC）を本論として扱う

## 基本ワークフロー

1. 参照資料を読み、対象章の位置づけ（第何部・第何章か）を確認する
2. 章テンプレート（概念/必要性/構成/失敗/日本注意点/語彙/要点）に沿って下書きする
3. 一次情報リンク（RFC、仕様書、公式文書）を最優先で補強する
4. 日本実務への接続（制度、運用、監査、利用者保護）を章内に必ず入れる
5. 用語統一（identity / identifier / credential など）を維持して仕上げる

## 出力ルール

- 言語: 日本語（技術用語・仕様名は英語可）
- 形式: `docs/bok/` 配下の Markdown
- 文章: 要約で終わらず、実務で使える判断観点・失敗例・チェック観点を含める

## Do / Don't

- Do: 章末に「要点まとめ」を入れる
- Do: 導入・運用・監査・事故対応までの接続を意識する
- Don't: 海外一般論のみで完結させる
- Don't: 既存資料の翻訳・抄訳のみで構成する
